## Overview

The Claude Code Developer Hub is a holistic framework for achieving rapid development velocity across multiple independent projects while maintaining world-class standards and best practices.

## Core Philosophy

```
Developer Velocity = (Intelligent Automation × Consistent Patterns × Deep Context) / Cognitive Load
```

## Best Practices Integration

### From Anthropic Prompt Engineering

1. **XML Structure Everything**
   ```xml
   <command_input>
     <raw_input>{{user_text}}</raw_input>
     <structured_intent>{{parsed_intent}}</structured_intent>
     <context_references>{{discovered_context}}</context_references>
   </command_input>
   ```

2. **Multi-Shot Examples**
   - Each command includes 3-5 examples
   - Examples cover edge cases
   - Progressive complexity in examples

3. **Explicit Reasoning**
   - Commands show their thinking
   - Decisions are transparent
   - Alternatives are considered

### From Your Developer Patterns

1. **Monorepo Architecture**
   ```
   [project]-xyz/        # Web platform
   [project]-ai/         # AI services (Mastra)
   [project]-apple/      # Native iOS
   [project]-docs/       # Documentation
   ```

2. **AI Service Pattern**
   ```typescript
   // Consistent Mastra implementation
   const agent = new Agent({
     model: runtimeContext.selectedModel,
     tools: { ...projectTools },
     instructions: contextualPrompt
   });
   ```

3. **Documentation-First**
   - Every project has CLAUDE.md
   - Architecture decisions documented
   - Version compatibility tracked

## Command Enhancement Patterns

### 1. Progressive Disclosure
```xml
<output_modes>
  <summary>3-5 key points for quick scan</summary>
  <detailed>Full analysis with examples</detailed>
  <expert>Deep technical implementation</expert>
</output_modes>
```

### 2. Validation Loops
```xml
<validation_checkpoint>
  <ask_user>
    "Based on my analysis, I understand you want to:
    1. {{understanding_1}}
    2. {{understanding_2}}
    
    Should I proceed with this approach?"
  </ask_user>
</validation_checkpoint>
```

### 3. Context Window Management
```xml
<context_strategy>
  <prioritization>
    <critical>Must be in context</critical>
    <important>Include if space allows</important>
    <reference>Can be summarized</reference>
  </prioritization>
  <compression>
    <technique>Semantic summarization</technique>
    <maintain>Key decisions and rationale</maintain>
  </compression>
</context_strategy>
```
